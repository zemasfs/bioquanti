Commands

      Jmol offers a comprehensive set of commands for visualizing and interacting with molecular models. These commands can be accessed primarily through the program’s main menu or through right-click interactions within the Jmol interface, including the web-based JSmol applet.

      However, scripting functionality via the console, available in both the installed program and the JSmol applet, unlocks a significantly broader range of capabilities. Scripting allows for the execution of sequential commands, enabling the generation of customized visualizations, calculations, animations, and even rudimentary programming using the Jmol scripting language. Comprehensive documentation for both scripting commands and the programming language is available on the developer’s website: Jmol - Documentation & Scripting. This website provides extensive examples and interactive links for further exploration.

      This chapter focuses on summarizing the essential console and script-based commands used for viewing, manipulating, importing/exporting, performing calculations on, and animating molecular models.

Creating Jmol Scripts

      For complex observations or animations, scripts can be created using any basic text editor. These scripts should be saved with the “.spt” file extension. Scripts can then be loaded as files or directly pasted into the console field for execution.

Jmol Command Syntax

      Jmol commands are case-insensitive and do not differentiate between singular and plural forms. Both “show group” and “Show groups” will produce the same results.

Adding Comments to Jmol Scripts

      When constructing scripts, it is recommended to include comments within the code for improved readability. Comments can be added using the hashtag symbol (#) placed at the beginning of a line following the desired command.

Molecular Model Databases and File Formats

      There are several online databases that provide access to molecular models for rendering and download in formats compatible with Jmol. Some prominent examples include:

PubChem: This database focuses on small molecules;

DrugBank: A resource for information on small molecule drugs and their targets;

RCSB Protein Data Bank (RCSB PDB: This database houses a vast collection of experimentally determined structures of biological macromolecules, such as proteins and nucleic acids;

IUPHAR/BPS Guide to PHARMACOLOGY: This resource provides access to information on both small molecules and macromolecules relevant to pharmacology.

      Within these databases, you can typically search for molecules of interest and download their structures in formats such as PDB, MOL, and SDF, which are all compatible with Jmol for visualization.

Controlling Molecular Representation Size

      The visual representation of a molecule in Jmol can be adjusted based on the size of its constituent atoms. For instance, the “spacefill” rendering method scales the atom radius according to the van der Waals radius of each atom.

      Command Examples:

spacefill on/off: Enables or disables spacefill rendering.
wireframe 0.5: Sets the wireframe thickness to 0.5 Angstrom (absolute value).
backbone 50: Sets the backbone thickness to 50/250 Angstrom, which equals 0.2 Angstrom (relative value).
cartoon 50%: Increases the cartoon thickness by 50% compared to the default setting.
spacefill 2: Sets the spacefill radius to 2/250 Angstrom, which is approximately 0.008 Angstrom (no decimal point indicates relative value).
spacefill 2.0: Sets the spacefill radius to 2.0 Angstrom (decimal point indicates absolute value).

Additional Notes:

      The provided commands demonstrate how to control the size of various representations. Jmol offers a wider range of visualization styles beyond those shown here.

      Some specific operations also allow tagging.

label binding site # label with the expression

label binding | site # tag with the expression, separated by lines ("|" is the "pipe" command)

      Jmol also allows to tag a specific atom, residue, interaction or reaction site, or some group for better identification and exploitation of the model. To do this, you can select the structure by select, followed by a proposed label. For example:

select pro31 # Proline selectio at position 31 in the protein primary sequence
label Proline-31 # labelling an specific Pro residue
set echo top left; echo "Model 1" # text insertion
select all; label on # atomic symbol and number for all atoms in the model

Selecting Atoms in Jmol

     General Atom Selection:

To select a general group of atoms, use the following mouse and keyboard combination:

Hold down the Shift key.
Left-click on the first atom in the desired group.
Drag the mouse while holding down the Shift key to encompass the rest of the group.

Nature:

all, none, protein, nucleic, dna, rna, carbohydrate, hetero, ligand, solvent, water ou hoh, ions, hem, amino

Chemical element:

  • By name: oxygen, nitrogen, carbon
  • By symbol: _N, _C, _Fe
  • By atomic number: elemNo=7, ou atomNo=5
  • By identifier number in the sequence (serial): atomNo<50

Parts of the molecule:

backbone, sidechain, :b (for chains), spine

Residue type

3-letter abbreviation of amino acids, acidic, basic, polar, neutral, charged, purine, pyrimidine, large, small, buried, surface, cyclic, hydrophilic, hydrophobic, aromatic, cystine, aliphatic, spine, base

Secondary structure of proteins

helix, sheet, turn

     The general commands for using the above expressions or combinations of these are, basically:

select, restrict, hide, display, delete

     Each expression must be enclosed in parentheses. You can also apply Boolean operations (or, and, but not, or =, <, >, <=, >=, !=).

      The restrict command is irreversible in nature, while the hide command is reversible during rendering. followed by the desired expression. Some examples:

      Here are some examples for the above commands:

Command restrict

*   restrict helix # only the helices of the secondary structure remain
*   restrict ligand # only the ligand
*   restrict not ligand # only proteins without prosthetic groups
*   restrict 42-60 # only the indicated residue sequence remains
*   restrict f6p # only the identified group remains (a linker, in this case)
*   restrict not(:a, :b) # strings other than "a" and "b" remain

Command select

  select Tyr
  select polar
  select all 
  select none
  select hem and :b # only the hem group of the "b" chain
  select hem and not iron :b # selects the heme group of the "b" chain, except the Fe atom in "b" chain
  select */2 # all atoms in model 2
select connected(oxygen) # all atoms linked to an oxygen
   select ser70 :A .ca # serine alpha carbon no. 70 of chain A
   select hetero and not solvent # heteroatoms except water
   select hetero and not ligand # heteroatoms except ligand
   select buried only # only residues from the protein core
   select helix or backbone # select helices or protein backbone
   select not helix # selects the other 2nd structures. protein (turn, sheet)
   select 62 and(sidechain or alpha) # sidechain or alpha carbon of residue 62
   select ser70.ca # alpha carbons of serine no. 70
   select ser70 :a.ca # serine alpha carbons no. 70 of chain "a"
   select (lys,arg) # only Lys and Arg residues
   select (lys,arg) and :b # Lys and Arg residues from chain "b"
   select 19-32 :y # residue sequence of chain "y"
   select asp47 # Asp residue at position 47
   select [asp]47 :y # residue asp47 from string "y"
   select (10 or 25) and backbone
   select resNo>14 and resNo<25 and *.CA # alpha carbons of residues 14 to 25
   select hoh.o/2 # oxygen from water molecules in model 2
   select (14,16,18,20) # the numbered residues
   select backbone and *:b # the backbone of chain "b"
   select backbone and not *:b # the backbone of all strings except "b"
   select not hidden...select displayed # everything that is not hidden
   select not selected # everything that was not previously selected
   select carbon and not selected # carbon atoms not previously selected

Special selections

within command

      This command allows a recursive selection, that is, a selection within a selection.

select within (5,(bca,amp)) # selects two ligands that are 5 Angstroms away from the selected group of atoms

select within(3.0, _Fe) # all atoms within 3 Angstroms of the iron atom

Command define

      Allows you to define a group of atoms for specific rendering or observation.

   defines Site within(3.0, _Cu) # 3 Angstroms away from copper atom
   defines PointX within(5.0,arg31) # 5 Angstroms away from Arg31
   defines neighbors selected # previously selected

Animated magnification (zoomto)

      This feature allows you to visualize in a temporally expanded way some parts of interest in the model, such as ligand interaction sites or prosthetic groups. The expression syntax is:

zoomto or zoomTo (atom/group expression) size
or...

zoomto time (atom/group expression) size

      Examples:

Zoom in by 3x, half a second at a time: zoomto 0.5 *3
Zoom in by 4x, half a second at a time: zoomto 0.5 400
Focus on a ligand at 2x magnification: zoomto 2(ligand) 0
Focus on a ligand at 4x magnification, half a second at a time: zoom to 0.5(ligand) *4

Surfaces

      Jmol allows you to render some types of surfaces around the model’s atoms. For macromolecules, it should be noted that rendering will take longer.

dots on/off # van der Waals surface
   dots only # only the vdw surface present (without the model)
   dots vanderwaals...or dots vdw
   ionic dots

isosurface on/off # molecular surface
   isosurface solvent # excluding solvent
   molecular isosurface # including solvent
   molecular isosurface 5 # surface at a given VDW radius
   isosurface sasurface # solvent accessible surface
   isosurface mep # "molecular electrostatic potential"
   isosurface map vdw # Van der Waals surface
   isosurface map mep # color mapping
   isosurface molecular map property PartialCharge # or temperature, vanderwaals

geosurface on/off # geodesic (curved) molecular surface
   geosurface only # exclusively the surface, without the model
   geosurface vdw
   ionic geosurface
   geosurface 5 # value in Angstrom
  
Note: To visualize the surfaces including the solvent, you can use the "set solvent on" command first.

Graphics

      Some two- and three-dimensional graphs can be generated using simple Jmol commands. To return to the original model, give the model 1 command.

   plot properties resno partialcharge # resno = residue number (plots after command to calculate surface charge - see above)
   plot properties phi psi resno # 3D plot with dihedral angles against no. of the residue
   plot rama...or plot ramachandran # Ramachandran plot for structure 2a. in proteins
  
   Other properties: hydrophobicity, mass, temperature, radius, occupancy, chaino

Creating and reading scripts

      Scripting allows the observation and analysis of a specific part of the model under study, by composing code snippets as command lines. In this way, the script resembles an algorithm with a succession of commands. A script can be loaded by calling the program’s files (File, Open), or even by dragging the file to the program’s screen area.

      The creation of the script can be done in a simple text file, saving it as **.spt. On the other hand, Jmol* has its own construction and testing environment, Script Editor, accessible through the main Menu. In this environment it is possible to open, create, check, and run a script. It is also valid to copy/paste some code snippet into Console or Jmol/JSmol, separating the commands with semicolons on successive lines. This way the Script Editor is simulated in the Console itself.

      It is also possible through Jmol to define an initial script to be run when the program opens. For this:

set defaultLoadScript "commands"

Mutation

      It is also possible to observe consequences of the mutation of specific residues in a model, such as weak interactions (electrostatic or hydrogen bonds, for example).

   mutate 33Lys # specifically changes position 33 of the primary sequence to Lysine
   mutate Ala34 ~W # change Ala to Trp
   mutate(1:3) ~GAL # changes the first three positions in the protein to Gly, Ala and Leu
