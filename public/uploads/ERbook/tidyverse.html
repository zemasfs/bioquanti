9  Tidyverse

      Tidyverse consiste num ecossistema de pacotes do R desenvolvido com foco em Data Science.

9.1 Guia de Estilos:

  • long names e explicativos
  • espaços entre operações;
  • sem espaços quando função “(…)” ;
  • introduzir espaços pra melhorar alinhamento (identação); pipe com espaço após dataset;
  • 2 espaços na linha de baixo se argumentos sucessivos (identação);
  • separação em seções comentadas: uso de # seção A

9.2 Racional de Análise

  • “main tools of data science: importing, tidying, transforming, and visualizing data”*
  1. Import: cria ou importa dos dados (import) - read.csv, read.table, tibble, tribble
  2. Tidy: arruma os dados; se importado, rearranja pra plots/stats (reshape) - pivot_longer, pivot_wider
  3. Transform: modifica variáveis - dplyr (filter, arrange, mutate, transmute, rename, relocate, groups_by, summarize, slice)
  4. Visualize: plota os dados - ggplot2, base, lattice
  5. Model: analisa e modela os dados (anova, lm, nls, stats, pacotes)
  6. Communicate: apresenta e compartilha a análise - rmarkdown, bookdown, blogdown, quarto

9.3 O ecossistema Tidyverse

      No tidyverse os comandos são concatenados pelo operador pipe, %>% (atualmente, |>). O pipe permite que a ação sucedente seja realizada com o resultado da anterior, evitando-se definir variáveis para cada uma.

9.3.1 Pacotes úteis do tidyverse

library(readr) # importação de database
library(magrittr) # uso do operador pipe
library(tibble) # conversão de data.frame pra manipulação
library(dplyr) # transformação de dataset (mutate/transmute, select, filter, slice, arrange, summarize, group_by)
library(ggplot2) # gráficos elegantes

9.3.3 O pacote dplyr e uso prático

      Em Ciência de Dados trabalha-se com conjuntos multifatoriais de dados colhidos em grande quantidade. Para esse tipo de situação, a extração de dados e de subconjuntos pode ser mais trabalhosa pela sintaxe convencional do R, e mesmo pelas funções de vetorização (família apply). Aí entram em cena alguns verbos utilizados no Tidyverse para simplificar as operações de extração de subconjuntos. O exemplo a seguir aplica alguns desses verbos a um dataset criado com poucos cliques.

library(tidyverse)
data <- read.table(header=TRUE, text='

n    t    x   y
2    10    0.46    1.74
3    25    0.47    2.44
4    50    0.49    3.19
5    100    0.52    4.31
6    150    0.54    5.14
7    200    0.563    5.88
8    300    0.58    7.13
9    400    0.59    8.15
10    500    0.61    9.10')

library(magrittr)
data2 <- data %>% # renomeia o dataset original
  select(t,y) %>% # seleciona as colunas t e y
  mutate(ycalc=y*1e-6) %>% # cálculo na coluna y
  filter(ycalc > 5e-6) %>% # filtra a coluna y
  arrange(desc(ycalc)) %>% # arranja em ordem decrescente
  rename(ycName=ycalc) # renomeia a coluna y

print(data2) # apresenta o resultado
    t    y   ycName
1 500 9.10 9.10e-06
2 400 8.15 8.15e-06
3 300 7.13 7.13e-06
4 200 5.88 5.88e-06
5 150 5.14 5.14e-06
