16  Carregamento de dados

16.1 Carregamento dos dados

      Há diversas formas de carregamento dos dados no R. Mas na prática, ou você 1) copia de uma planilha os dados obtidos na bancada de laboratório, por exemplo, e cola no R, ou você 2) monta uma tabela na unha, mesmo, ou 3) importa o arquivo dessa planilha.

16.2 Colando os dados de uma planilha ou montando a tabela “na unha” !

     Para colar os dados copiados de um Excel (Microsoft) ou Calc (Libreoffice), basta copiar o trecho desejado e colar no espaço mencionado na função abaixo, entre os (’ … ’):

# Colando os dados de uma planilha
meus.dados <- read.table(header=T, text='
dados # colar aqui os dados
')

      Traduzindo….

  1. O símbolo “<-” indica que você vai atribuir aos dados copiados da planilha o nome “meus.dados”. Isso é feito pra se trabalhar depois com esse objeto do R, pra plotagem e análises. Pode-se utilizar o sinal de igualdade =, embora haja discussões a respeito. Se por um lado o sinal de igualdade refere-se mais a atribuição de valores do que de objetos, por outro possui digitação mais sucinta e menos ambígua em caso de erro tipográfico.
  2. A função read.table é quem manda o R fazer essa atribuição
  3. O que está dentro dos parênteses é o que se chama argumentos da função. No caso, os argumentos são para a) manter os nomes das colunas (header = T; T = TRUE), e 2) colocar o material copiado no espaço (text = ´…´).

  Pasme, mas….pra montar uma tabela na unha o comando é o mesmo. Basta separar os elementos com um espaço:

# Criando a tabela de dados
meus.dados <- read.table(header = T, text  = '
  A B C
  1 2 3
  4 5 6
  ')

16.3 Importando os dados de uma planilha

      Nada complicado, só a função é que muda. Você pode importar do MS Excel, do Libreoffice Calc, ou de outra qualquer. Para os dois primeiros:

16.3.1 Importando do Excel

      Basta você carregar uma biblioteca (ou pacote) antes, chamada readxl. É facinho, veja:

library(readxl)

      Agora você pode importar um arquivo do Excel, só observando que, a menos que você queira complicar mais um pouquinho a linha de comando, deve deixar apenas uma aba presente.

meus.dados <- read_excel("dados.xlsx")

16.3.2 Importando de outras planilhas (Libreoffice)

      Também é possível importar os dados de planilhas como o Calc do Libreoffice. Pra isso basta salvar o arquivo na planilha como .CSV (“comma separated value”). E não precisa instalar nenhuma biblioteca adicional.

meus.dados <- read.csv("dados.csv")

      Alternativamente, pode-se selecionar o arquivo CSV a partir de uma janela separada, como em:

read.csv( file.choose( ) )

      A importação de arquivos pode esbarrar num erro comum de separação de valor decimal. Nesse caso deve-se atentar para o tipo de símbolo decimal empregado pela planilha (editor de texto, Excel, Calc, etc). Exemplificando:

dados <- read.csv("dados.csv", dec = ",")

      Também é possível a importação de dados no formato TXT pelo uso da função scan. Nesse caso a importação dá-se exclusivamente por dados numéricos e como uma matriz.

16.3.3 Exemplos de datasets

      A observação, análise e produção de gráficos de conjuntos de dados pode ser realizada com datasets empregados normalmente para exemplificação de comandos e funções do R. Dois grandes conjuntos são normalmente utilizados, um conjunto interno que já vem na instalação do R, e outro, pertencente ao pacote datasets. Para identificar os conjuntos de dados presentes:

data()

16.4.1 Observação com o pacote explore

      O pacote explore permite uma descrição mais detalhada ainda de um dataset. É melhor empregado para conjuntos mais complexos, onde se deseja extrair informações exploratórias (Exploratory Data Analysis - EDA), com tabelas, sumários e gráficos. Ilustrando seu uso para o dataset ‘iris’ :

library(explore)
explore(iris)

# Obs: para visualizar o resultado deve-se carregar o trecho no `R`, já que a saída é de significativa complexidade gráfica.

16.5 Convertendo os dados importados para um objeto do R

Uma função bastante útil após a importação de um *dataset* é sua conversão para um objeto que possa ser trabalho no `R` para recriação, como ocorre no compartilhamento de um *script*, por exemplo. Ilustrando:
meus.dados <- read.csv("dados.csv")
dados <-dput(meus.dados)

      Também é possível utilizar o comando dput para a conversão de dados criados no R, como em:

# Criando a tabela de dados
meus.dados <- read.table(header = T, text  = '
  A B C
  1 2 3
  4 5 6
  ')

dados <- dput(meus.dados); dados
structure(list(A = c(1L, 4L), B = c(2L, 5L), C = c(3L, 6L)), class = "data.frame", row.names = c(NA, 
-2L))
  A B C
1 1 2 3
2 4 5 6

      Ao se atribuir uma variável ao segmento “structur(list…)”, o objeto pode ser compartilhado para uso em outro computador, por exemplo, sem riscos de ocorrer erros por distintas tipografias.

      Também é possível exportar essa estrutura de dados como um arquivo TXT para compartilhamento:

write.csv(dados, "meus.dados.csv")
