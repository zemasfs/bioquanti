3  Controle de versões

3.1 Criando um projeto no RStudio para versionamento de código

      O controle de versões faz parte da logística em Pesquisa Reproduzível (Vuorre and Curley 2018). Basicamente, esse controle permite o versionamento do código produzido para um documento ou projeto no R ou RStudio. Nesse último a interface gráfica facilita o monitoramento de versões. A grande vantagem por detrás do controle de versões é a possibilidade de recuperar uma versão eficiente do documento, sem que se precise salvá-lo a todo momento com nomes diferentes como forma de proteção.

      Segue um pequeno tutorial para a implementação do versionamento de códigos pelo Terminal para instalação de arquivos, e RStudio para uso do sistema.

  1. Crie o projeto em File -> New Project;
  2. Escolha New Directory;
  3. Escolha Quarto project;
  4. Digite o nome do diretória a criar;
  5. Selecione a caixa Create a git repository;
  6. Escolha se deseja um editor de texto simples ao iniciar ou um editor visual (use visual markdown editor).

3.2 Configurando GitHub

3.2.1 Instalando e configurando o Git no PC:

      Num Terminal, digite:

  1. sudo apt-get update
  2. sudo apt-get install git
  3. git version ; conferindo a instalação
  4. git config –global user.name “login de usuário” ; ex:jaime
  5. git config –global user.email “email do usuário” ; ex: jaime@gmail.com
  6. git config user.name ; conferindo username
  7. git config user.email ; conferindo email

      Alternativamente, conferindo ambos:

  1. git config –global –list

3.3 Criando um repositório no Github

  1. Entra no Github;
  2. Vá em Your Repositories (menu que abre no canto superior direito);
  3. Clicar em New

3.3.1 Definindo um Token para acesso ao repositório

  1. No Github, vá em:
  • Settings
  • Developer Settings
  • Personal access tokens
  1. Escolha uma das opções (ex: “Tokens classic”)
  2. Generate new token
  • Escreva uma nota sobre o token;
  • Coloque um período de expiração;
  • Selecione repo para os repositórios

  1. Clique em Generate token

  2. Copie essa senha para a próxima etapa, a de linkar o projeto local no RStudio para acesso remoto no Github.

3.3.2 Linkando o repositório criado ao RStudio

  1. Cria o projeto no RStudio, e com Version Control
  2. Vá à pasta criada e digite os comandos que a página do Github oferece, na sequência:
  • echo “# nome.do.repositório” >> README.md
  • git init
  • git add README.md
  • git commit -m “first commit”
  • git branch -M main
  • git remote add origin https://github.com/nomeDaContaGit/NomeRepositorioGit.git ; esse é o link oferecido pelo próprio Git
  • git remote add origin
  • digite a senha (token) obtida anteriormente no site do Github

Obs: - Um commit nada mais é do que a imagem das alterações locais realizadas em um projeto, ou seja, um registro de suas alterações. A cada commit são geradas remotamente no Github uma versão diferente do projeto, e que pode ser recuperada em caso de falhas, por exemplo. - Pode-se colar num Terminal as linhas globalmente, executando-as uma única vez:

echo "# nomeDoRepositorio" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/nomeDoUsuario/nomeDoRepositorio.git
git push -u origin main

3.4 Sincronizando o projeto do RStudio com o repositório no Github

  1. Reinicie o RStudio para aparecer a aba Git (se já não apareceu antes);
  2. Faça alguma modificação no arquivo do projeto (md, rmd, qmd).
  3. Salve o arquivo e/ou renderize (deverá aparecer arquivo na aba Git que foi editado localmente);
  4. Selecione os arquivos da aba Git para sincronização remota (um bom e velho clique + Ctrl-A seleciona todos);
  5. Clique em Commit, e digite qualquer observação no campo;
  6. Clique em Commit da nova janela;
  7. Clique em Push, para envio à nuvem;
  8. Digite o username para acesso ao Github;
  9. Digite o token obtido na seção anterior (senha)
  10. Feche as janelas.

3.5 Clonando um repositório do Git como projeto no RStudio

  1. Crie o repositório no Git;
  2. Crie o projeto em File -> New Project;
  3. Escolha Version Control;
  4. Selecione Git;
  5. Cole o caminho do link para o repositório (Repository URL);
  6. Crie um diretório para o projeto (Create Project);
  7. Clique em Create Project.

3.6 Deletando um repositório

  1. Entra no repositório no Git;
  2. Vá em Settings do repositório (menu superior), e não geral, que fica no canto superior direito;
  3. Lá embaixo tem a opção de delete.
Vuorre, Matti, and James P Curley. 2018. “Curating Research Assets: A Tutorial on the Git Version Control System.” Advances in Methods and Practices in Psychological Science 1 (2): 219–36.
